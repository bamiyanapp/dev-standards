# Goal

Claudeは「ユーザーの依頼を品質基準を満たした状態で完了させる」ことを目的とする。

途中成果ではなく、完了条件を満たすまで作業を継続する。

# Agent Principle

Claudeは「単に指示を実行する」のではなく、
Goal達成まで自律的にObserve→Plan→Act→Verify→Reflectを繰り返す。

各ループで現在状態(State)を確認し、
最小の次アクションを選択する。

新たな知見や再利用可能なパターンを得た場合は、
SkillsまたはCLAUDE.mdへの反映を検討し、
継続的に開発プロセスを改善する。

Reflectの結果は次のObserveに反映し、
Goal達成までループを継続する。

# Execution Loop

以下を繰り返す。

1. 現状を観測する
2. Goalとの差分を確認する
3. 次の最小タスクを決定する
4. 実行する
5. Verifierを実行する
6. 必要なら修正する
7. Goal達成まで繰り返す

Execution Loopは開発全体の抽象的な制御ループを示す。各ステップの具体的な実施内容は各Skillに委譲する。

# State

Claudeは開始時および中断復帰時に以下を把握する。

- 現在のブランチ
- git status
- 未コミット差分
- 直前の作業内容
- Goalに対する現在の進捗

# Reflection

完了前に確認する。

- より簡潔・安全な実装方法はないか
- 重複コードはないか
- Skillへ追加すべき知見はないか
- CLAUDE.mdを改善すべき点はないか
- プロジェクト固有ルールへ反映すべきか

# Skills (専門手順)

具体的な実行方法や手順は、以下のSkillに責務を委譲し、CLAUDE.mdには記述しない。
タスク遂行にあたっては、以下のSkillを適宜呼び出して使用すること。

- **development-loop**: 実装とテストの基本開発ループ
- **git-workflow**: ブランチ作成からpushまでのGit操作手順
- **verifier**: 静的チェック、コードレビュー、Goal達成確認を含む品質保証プロセス
- **commit**: コミット作成前の条件確認と必須項目のチェック
- **code-review**: 差分の自己レビュー（品質、バグ、テスト網羅等の確認）
- **git-conventions**: ブランチ命名規則およびコミットメッセージ（subject/body）フォーマットの詳細
- **safe-bash-commands**: ハングや対話プロンプトを回避する安全なコマンド実行方法

# 最後に

Claudeはタスクを直列の手順としてではなく、
Goal達成のための反復ループとして扱う。

途中で新しい情報や失敗が判明した場合は、
当初の計画を固定せず、
Observe→Plan→Act→Verify→Reflectを繰り返し、
最適な手段へ適応する。
