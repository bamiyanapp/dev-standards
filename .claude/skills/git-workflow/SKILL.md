# Git Workflow

作業開始

1. main更新
2. 作業ブランチ作成

作業終了

1. main取り込み
2. 競合解消
3. push
4. PR（MR）作成（ユーザーへ確認せず自動で行う。本項目自体が事前承認であり、都度作成の可否を尋ねない。
   同一の作業ブランチに対応するPRが既に存在する場合は新規作成せずpushのみで更新する。
   承認・マージの扱いは下記「PR（MR）承認・マージ禁止（強制）」を参照）
5. main最新化

ブランチ命名は
git-conventions Skillを使用する。

## PR（MR）承認・マージ禁止（強制）

Claudeは以下を一切行ってはならない。

- `gh pr review --approve` 等によるPR（MR）の承認
- `gh pr merge` 等によるPR（MR）のマージ・自動マージ設定の変更
- レビュー結果として「承認（approve）」を選択するコメント・ステータスの付与

PR（MR）の作成（`gh pr create`）およびコメント投稿までは許可される範囲だが、その先の承認・マージ判断は必ず人間（ユーザーやCI/自動マージの仕組み）に委ねること。ユーザーから明示的に「このPRを承認して」「マージして」と指示された場合でも、まず承認・マージの意図を1文で確認してから実行する。

この事前承認は上記「作業終了」フローにおけるPR作成・コメント投稿のみに適用され、承認・マージには一切及ばない。

リポジトリ固有のCI／自動マージの仕組みがある場合は、各リポジトリのCLAUDE.mdを別途参照する。
