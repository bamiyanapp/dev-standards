# Verifier

目的

成果物がGoalを満たしていることを保証する。

Verifierは以下で構成される。

1. 静的チェック
2. コードレビュー
3. Goal確認

失敗した場合は修正後、
Verifier全体を再実行する。

Verifierが成功するまで

- コミット
- 完了報告

を行ってはならない。

## 静的チェック

対象パッケージすべてで

- lint
- test
- build

を実行する。

エラーがあれば修正して再実行する。

## コードレビュー

code-review Skillを使用する。

レビュー観点

- 要件
- バグ
- テスト
- 可読性
- 将来の保守性
- セキュリティ

## Goal確認

以下を満たすこと。

- 要件達成
- lint成功
- test成功
- build成功
- code-review成功
- コミット作成可能
